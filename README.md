## サービスへの想い（作成背景）

ゼミ活動でのチーム開発になります。とてもアバウトなのですが、社会貢献性のあることをしたいなと考えたとき、GX が思い浮かびました。GX を促進させるには、何が必要かを考えたとき、炭素排出量を減らす・省エネを促進することが必要と考え、二つの事を行うためのアプリケーションを開発しました。

<br>

## GX 要素について(どうやって)

- 地下鉄利用促進
  → 公共交通機関では地下鉄が一番 CO2 排出量が少ないので、自動車から地下鉄へのシフトを促すことで CO2 削減につながると考えた。

- 省エネ対策
  → 駅などでの、広い土地では、構内のすべての状況を把握はできない。そこで、駅利用者から「待合室が暑い」・「便座が温かすぎる」など口コミを得ることによって、省エネにつなげられると考えた。

<br>

## 機能紹介（GX 実現のためのアプローチ）

- 口コミの閲覧・投稿
- 駅から一定距離以内に、デバイスがあれば駅への投稿が可能になる。
- 現状寄せられている口コミ全体の閲覧（路線と最寄り駅で絞り込みが可能）

- CO2 削減量の確認・記録
- 目的地までの移動距離を、自動車で移動したときの CO2 排出量と、地下鉄で移動したときの CO2 排出量を算出し、差分で CO2 削減量が確認できる。
- 駅を通過したかどうかを判定し、CO2 削減量を記録できる。

- ポイント
- 各駅に寄せられた口コミを、駅員が GX に貢献したか判定(クラウドのようなものを実装)し、GX 貢献ユーザーにポイントを付与できる(口コミ書くだけじゃ駄目で、貢献性が必要)
- 移動を記録し、出発駅・到着駅の 2 つの駅の通過を確認できたら、対応した CO2 削減量をポイントとして受け取れる
