## サービスへの想い（作成背景）

とてもアバウトなのですが、社会貢献性のあることをしたいなと考えたとき、GX が思い浮かびました。GX を促進させるには、何が必要かを考えたとき、炭素排出量を減らす・省エネを促進することが必要と考え、二つの事を行えるアプリケーションを開発しました。

## ターゲット

- 京都市民<br/>
  → 京都で学生生活を送っており、まずは京都市ユーザを対象にしようと思いました。

<br>

## GX 要素について(どうやって)

- 地下鉄利用促進<br/>
  → 公共交通機関では地下鉄が一番 CO2 排出量が少ないので、自動車から地下鉄へのシフトを促すことで CO2 削減につながると考えました。

- 省エネ対策<br/>
  → 駅などでの、広い土地では、構内のすべての状況を把握はできません。そこで、駅利用者から「待合室が暑い」・「便座が温かすぎる」など口コミを得ることによって、省エネに繋げられると考えました。

<br>

## 機能紹介（GX 実現のためのアプローチ）

- 口コミの閲覧・投稿<br/>
  → 駅から一定距離以内に、デバイスがあれば駅への投稿が可能になります。<br/>
  → 現状寄せられている口コミ全体の閲覧（路線と最寄り駅で絞り込みが可能）

- CO2 削減量の確認・記録<br/>
  → 目的地までの移動距離を、自動車で移動したときの CO2 排出量と、地下鉄で移動したときの CO2 排出量を算出し、差分で CO2 削減量が確認できます。<br/>
- 駅を通過したかどうかを判定し、CO2 削減量を記録できます。

- ポイント<br/>
  → 各駅に寄せられた口コミを、駅員が GX に貢献したか判定(クラウドのようなものを実装)し、GX 貢献ユーザーにポイントを付与できます(口コミ書くだけじゃ駄目で、貢献性が必要)<br/>
  → 移動を記録し、出発駅・到着駅の 2 つの駅の通過を確認できたら、対応した CO2 削減量をポイントとして受け取れます

## 今後の展望（サービス&個人）

- 目的地の選択肢の変更<br/>
  → 現状、駅から駅まででの、CO₂ 削減量の確認になってしまっています。理想は、現在地から目的地（観光地,etc...）で距離を取得し、その中で地下鉄を利用したら、CO2 がこのくらい削減できるという表示を行いたいです。

- 駅員側（口コミ確認・付与）画面の分離化、UI<br/>
  →「機能紹介（ポイント）」で、クラウドのようなものを実装と書いてあるが、現状 UI がシンプルすぎる。また、ログイン画面から駅のログインを行えるようになってしまっているので、別のサービスを立ち上げる必要があると考えています。

- デプロイ（インフラ）<br/>
  → ここは、自身の経験を積むためにも、挑戦してみたい。（時間あれば）

- モバイルアプリ化<br/>
  → デスクトップアプリとして開発したが、ユーザーは移動の記録、口コミの投稿など、アプリじゃないと行わない。<br/>
  使われるサービスになるためにもモバイル化を実装していきたい。
